# Palette Brand Guidelines

**Version**: 1.0  
**Date**: 2026-02-01  
**Status**: Production-Ready

---

## Visual Identity

### Glyph: "The Builder's Palette"

**Concept**: Programmer/builder figure holding painter's palette with 8 colored spots.

**Usage**:
- ✅ GitHub avatar
- ✅ README header
- ✅ Documentation
- ✅ Social media
- ❌ Do not modify colors
- ❌ Do not separate palette from figure
- ❌ Do not use partial palette (all 8 colors required)

**Minimum Size**: 32x32px (must remain legible)

---

### Usage Rules

**Do**:
- ✅ Use agent colors in documentation sections
- ✅ Use colors for workflow diagrams
- ✅ Use colors for status indicators
- ✅ Maintain WCAG AA contrast

**Don't**:
- ❌ Change hex values
- ❌ Use colors for non-agent purposes
- ❌ Mix with conflicting color schemes

---

## Typography

**Code/Technical**: Monospace (JetBrains Mono, Fira Code, system)  
**Prose**: Sans-serif (Inter, System UI, Helvetica)

**Hierarchy**:
- H1: 32px bold
- H2: 24px semibold
- H3: 20px semibold
- Body: 16px regular
- Code: 14px monospace

## Message Patterns

### Errors
`[Agent] → [What happened] → [Why] → [What to do]`

### Success
`[What succeeded] → [Evidence] → [Next step]`

### Contributions
`[Thank you] → [What you contributed] → [Impact]`

## Accessibility

- All colors meet WCAG AA standards
- Color-blind friendly palette
- Monospace-friendly typography
- Readable at small sizes (32px minimum)
