## KEY FRAMEWORKS TO USE

### 4-Question Framework (Use in Demo Scenario)
1. Is there a pattern?
2. Is there data?
3. Is it repetitive or high-volume?
4. Can you measure success?

### Agent Workflow Design (Use in Demo Scenario)
1. **Input**: What does the user provide?
2. **Retrieval**: What data does the agent need?
3. **Generation**: What does the agent produce?
4. **Human-in-the-loop**: Where does the user review and edit?

### Pilot Scoping (Use in Demo Scenario)
- **Duration**: 12 weeks
- **Users**: 5-10 users
- **Success criteria**: 80%+ usage, 50%+ time savings, 4.5+ satisfaction
- **Kill criteria**: <50% usage, <30% time savings, <4.0 satisfaction

### Success Metrics (Use in Q&A)
- **Usage metrics**: Daily/weekly active users, frequency, retention
- **Efficiency metrics**: Time saved, tasks completed, error reduction
- **Business outcome metrics**: Win rate, sales cycle, customer satisfaction

## COMMON MISTAKES TO AVOID

### 1. Talking Too Much About Theory
- Don't: "AI agents are the future of work..."
- Do: "I built Palette â€” here's how it works and what it's shipped..."

### 2. Not Asking Questions
- Don't: Just answer their questions and wait for next steps
- Do: Ask intelligent questions about product, customers, roadmap, role

### 3. Being Vague About Outcomes
- Don't: "Palette helps me work faster..."
- Do: "Palette reduced interview prep time from 4-6 hours to 20 minutes (15X faster)"

### 4. Not Showing Customer Empathy
- Don't: "I'd build an AI agent that does X..."
- Do: "I'd start with discovery: What does success mean to you? What's the current process?"

### 5. Overselling AI Capabilities
- Don't: "AI can automate everything..."
- Do: "AI can generate a first draft, but you'll need to review and finalize it"
